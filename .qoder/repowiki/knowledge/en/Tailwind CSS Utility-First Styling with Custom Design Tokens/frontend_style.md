## System Overview

This Blazor Interactive Server application uses **Tailwind CSS v3.4.19** as its primary styling framework, following a utility-first methodology. The build pipeline is managed through npm scripts that compile source CSS into production-ready output.

## Key Files and Packages

### Core Configuration
- **`tailwind.config.js`** — Tailwind configuration defining content paths (all `.razor` files under `Components/`, plus `wwwroot/**/*.html` and `*.js`), dark mode strategy (`class`-based), and custom font families.
- **`Styles/app.css`** — Source CSS file containing Tailwind directives (`@tailwind base/components/utilities`) and custom layer definitions for base styles, animations, and reusable component classes.
- **`wwwroot/app.css`** — Compiled output (~2472 lines) generated by the Tailwind CLI from `Styles/app.css`.
- **`package.json`** — Defines npm scripts: `build:css` (one-shot compile) and `watch:css` (file watcher). Dev dependencies include `tailwindcss`, `postcss`, and `autoprefixer`.

### Theme Management
- **`wwwroot/js/theme.js`** — JavaScript module exposing `window.themeManager` with methods `setTheme()`, `getTheme()`, and `initTheme()`. Persists user preference to `localStorage` and respects `prefers-color-scheme` media query as fallback. Applies/removes `.dark` class on `<html>` element.

### Component-Level Styles
- **`Components/Layout/MainLayout.razor.css`** — Blazor CSS isolation for layout structure (sidebar gradient, top-row styling, responsive breakpoints at 640px).
- **`Components/Layout/NavMenu.razor.css`** — Isolated styles for navigation menu including Bootstrap-style navbar toggler, nav item hover states, and sidebar scroll behavior.

## Architecture and Conventions

### Build Pipeline
CSS compilation follows this flow:
```
Styles/app.css (source) → tailwindcss CLI → wwwroot/app.css (compiled)
```
The `tailwind.config.js` content array scans all Razor component files, ensuring only used utility classes are included in the final bundle (purge optimization).

### Dark Mode Strategy
Dark mode is implemented via the `class` strategy (not `media`). The `theme.js` script runs immediately on page load (`initTheme()` auto-invoked) to prevent flash-of-unstyled-content. Components toggle dark mode by calling `themeManager.setTheme(bool)` via JS interop.

### Design Tokens
Custom design tokens are defined in `tailwind.config.js` under `theme.extend`:
- **Font families**: `sans` stack prioritizes `Outfit`, then `Inter`, falling back to system fonts. `mono` stack uses `Fira Code` / `JetBrains Mono`.
- **No custom color palette extension** — relies entirely on Tailwind's default color scale (slate, blue, emerald, rose, violet, etc.).

### Custom Utility Classes
Defined in `Styles/app.css` within `@layer utilities`:
- **`.glass-card`** — Reusable card component: white background, slate border, rounded-xl, shadow-sm, p-6 padding.
- **Animation utilities**: `.animate-slide-up`, `.animate-fade-in`, `.animate-fade-in-up` — each with cubic-bezier easing `(0.16, 1, 0.3, 1)` for smooth entrance effects.

### Base Layer Overrides
The `@layer base` block sets global defaults:
- `html, body`: `bg-gray-50`, `text-slate-800`, `font-sans`, `antialiased`, `scroll-smooth`
- Selection styling: `bg-blue-500/20` background, `text-blue-700` foreground
- Custom scrollbar: 6px width, transparent track, `bg-slate-300` thumb with hover transition

### Component Styling Patterns
Razor components use inline Tailwind utility classes extensively. Common patterns observed:
- **Cards**: `bg-white border border-slate-200 rounded-xl shadow-sm` or the `.glass-card` utility
- **Buttons**: `px-5 py-2.5 bg-blue-600 hover:bg-blue-700 text-white text-sm font-bold rounded-xl shadow-md transition-all active:scale-95`
- **Typography**: `text-xs font-bold text-slate-800` for body, `text-[10px] font-black uppercase tracking-wider` for labels
- **Modals**: Fixed overlay with `bg-slate-900/60 backdrop-blur-sm`, content with `animate-slide-up`
- **Responsive**: Mobile-first with `md:` and `lg:` breakpoints; grid layouts use `grid-cols-1 md:grid-cols-3`

### Legacy CSS Isolation
Two layout components (`MainLayout.razor.css`, `NavMenu.razor.css`) use Blazor's CSS isolation feature with `::deep` selectors for penetrating child component boundaries. These contain legacy gradient backgrounds and Bootstrap-inspired navbar patterns that coexist with the Tailwind system but are not the primary styling approach for page-level components.

### Inline Component Styles
Some Razor pages (e.g., `Home.razor`, `BillDetails.razor`) define component-scoped `<style>` blocks for keyframe animations (`float-slow`, `float-medium`, `float-fast`, `fade-in`) and custom scrollbar utilities. This is an accepted pattern for animations not covered by the global utility layer.

## Rules Developers Should Follow

1. **Always use Tailwind utility classes** in Razor component markup for layout, spacing, colors, and typography. Avoid writing new CSS rules unless absolutely necessary.

2. **Add custom utilities to `Styles/app.css`** within the appropriate `@layer` directive (`base`, `components`, or `utilities`). Never add arbitrary CSS outside these layers.

3. **Use the `.glass-card` utility** for standard card containers instead of repeating `bg-white border border-slate-200 rounded-xl shadow-sm p-6`.

4. **Follow the established color semantics**: `blue-600` for primary actions, `emerald-600` for success/positive states, `rose-600` for danger/negative states, `slate-*` for neutral text and borders.

5. **Apply animation utilities** (`.animate-fade-in`, `.animate-slide-up`, `.animate-fade-in-up`) for modal entrances and page transitions. For custom animations, define keyframes in a component-scoped `<style>` block.

6. **Run `npm run watch:css`** during development to automatically recompile Tailwind when Razor files change. Run `npm run build:css` before deployment.

7. **Use `themeManager.setTheme()` via JS interop** to toggle dark mode. Do not manipulate the `.dark` class directly from C# code.

8. **Prefer Tailwind's responsive prefixes** (`sm:`, `md:`, `lg:`, `xl:`) over custom media queries. The project uses standard Tailwind breakpoints (640px, 768px, 1024px, 1280px, 1536px).

9. **Keep font usage consistent**: Use `font-sans` (Outfit/Inter) for UI text and `font-mono` (Fira Code/JetBrains Mono) for numeric values, especially currency amounts (`font-mono font-black`).

10. **Do not modify `wwwroot/app.css` directly** — it is a generated file. All changes must go through `Styles/app.css` and the Tailwind build process.
